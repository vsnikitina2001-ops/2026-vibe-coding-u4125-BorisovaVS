```text
Создай Telegram-бота на Python с использованием библиотеки python-telegram-bot.

Функционал бота:
1)Умный список дел:
- Пользователь может добавлять задачи с текстом
- Возможность указать приоритет (низкий / средний / высокий)

- Просмотр списка покупок
3) Планировщик уборки:
- Разделение задач по зонам (кухня, ванная, спальня, прихожая)

-Возможность отмечать выполненные пункты
-Напоминания о запланированной уборке
-Отображение прогресса выполнения
4)Напоминания о важных датах:
-Добавление событий (дни рождения, праздники, личные даты)
-Хранение даты и названия события
-Напоминание заранее (например за 1 день)
-Просмотр списка событий

Команды бота:
/start — приветствие и описание возможностей
/help — список команд
/tasks — управление списком дел
/add_task — добавить задачу
/shopping — список покупок
/add_item — добавить покупку
/cleaning — план уборки
/add_cleaning — добавить задачу уборки
/events — список событий
/add_event — добавить событие

Как бот должен отвечать:
-Короткие и понятные сообщения

-Напоминания отправлять автоматически в чат пользователя
-При ошибках выводить понятное сообщение

Какие данные хранить:
-Пользователи (user_id)
-Список задач (текст, приоритет, статус, дата/время)
-Список покупок (название, категория, статус)
-Задачи уборки (зона, чек-лист, статус)
-События (название, дата, напоминание)

Хранить данные в JSON-файле или SQLite базе данных.

Требования:
-Бот должен быть простым и понятным
-Код должен быть хорошо прокомментирован
-Использовать файл для хранения данных (JSON или SQLite)
-Добавить обработку ошибок (try/except)
-Структурировать код (разделить логику по функциям)

Ограничение доступа:
-Бот должен работать только для заранее заданного списка пользователей (whitelist)
-Список разрешённых user_id хранится в конфигурации (например, в .env или отдельном файле)
-Если пользователь не входит в список — бот должен отправлять сообщение: "У вас нет доступа к этому боту"
-Все команды должны проверять доступ пользователя перед выполнением

Создай:
-Файл bot.py с кодом бота
-Файл requirements.txt с зависимостями
-Файл README.md с инструкцией по запуску
-Файл .env.example для примера конфигурации
```
